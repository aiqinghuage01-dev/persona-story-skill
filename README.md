# 人设故事

> 员工视角讲老板，人设型短视频口播文案

## 项目背景

这是清华哥（AI流量获客培训讲师）给学员打造的短视频文案生产skill之一。

**学员画像**：全国实体老板（餐饮、教培、工厂、大健康、设计、美业等），大多数不会写文案，不懂短视频，需要AI直接出成品。

**这个skill的定位**：人设型内容。帮老板用"员工视角讲老板"的故事结构，建立粉丝信任。属于"短视频文案工具箱"的其中一个模板。

**设计原则**：
1. 老板只需要提供一个故事素材，AI自动出完整文案
2. 知识和逻辑分离——范例、结构、规范各自独立，改一处不影响其他
3. few-shot驱动——7个行业范例决定输出质量，不靠长prompt约束

**模板来源**：从一条真实爆款视频（"我的老板是厂二代"）中拆解出七段结构，然后在6个不同行业中验证可复用性。

## 这个skill做什么

输入：老板的知识库 + 一个故事素材
输出：600-900字可直接读稿的短视频文案

## 文件说明

| 文件 | 谁看 | 干什么 |
|---|---|---|
| `SKILL.md` | AI执行 | 生成逻辑，AI执行的唯一指令 |
| `知识库/模板结构.md` | AI参考 + 开发者 | 七段结构定义，改结构改这里 |
| `知识库/写作规范.md` | AI参考 + 开发者 | 文案铁律 + 反面案例，改风格改这里 |
| `知识库/行业范例/*.md` | AI参考 + 开发者 | few-shot范例，加行业加这里 |
| `evals/test_cases.md` | 开发者 | 测试用例，改完skill跑一遍验证质量 |
| `VERSION` | 开发者 | 当前版本号 |
| `CHANGELOG.md` | 开发者 + AI | 迭代记录，含设计决策说明 |
| `README.md` | 开发者 + AI | 你正在看的这个文件 |

## 范例覆盖行业

| 序号 | 行业 | 覆盖学员类型 |
|---|---|---|
| 01 | 服装厂 | 制造业（原版标杆，模板从这条拆出来的） |
| 02 | AI培训 | 教育培训（清华哥自用版本） |
| 03 | 铁锅炖鱼 | 餐饮/门店 |
| 04 | 橱柜工厂 | 制造业toC |
| 05 | 乒乓球教练 | 教育教培 |
| 06 | 养生馆 | 大健康/中医 |
| 07 | 工程设计 | 设计/轻资产toB |

行业选择依据：基于线下课568名学员的行业分布数据，覆盖Top5行业（教育教培15人、大健康25人、工业/制造15人、服务业14人、设计9人）。

## 加新范例的规范

1. 文件名格式：`序号_行业_角色.md`（如 `08_美业_美甲店.md`）
2. 文件头部写明：行业、视角、产品植入
3. 必须是完整文案（不是大纲），600-900字
4. 严格遵循 `模板结构.md` 的七段结构
5. 加完后在 `SKILL.md` 的匹配规则里补上对应行业
6. 在 `CHANGELOG.md` 里记一笔

## 协作规范

- 改了任何文件 → `CHANGELOG.md` 加一条，写日期、写谁改的、写改了什么
- 改了结构或规范 → `VERSION` 中间那位数+1（如 1.0.0 → 1.1.0）
- 只加了范例 → `VERSION` 最后一位+1（如 1.0.0 → 1.0.1）
- 改完之后 → 用 `evals/test_cases.md` 里的用例跑一遍，确认输出质量没掉
